# Testnet to Mainnet Migration

## Background

The StorX project transitioned from Testnet to Mainnet as part of the platform roadmap.

My node was originally deployed on Testnet in June 2022 and later migrated to Mainnet.

## Migration Process

* Reviewed official migration documentation.
* Updated server environment.
* Applied required StorX node updates.
* Performed migration procedures.
* Verified node synchronization.
* Confirmed dashboard visibility and node health.

## Challenges

### Dashboard Delay

Following migration, the dashboard did not appear immediately.

Resolution:

* Performed Ubuntu updates.
* Rebooted the server.
* Allowed additional synchronization time.

### Reputation Recovery

At one point the reputation score dropped significantly below the expected value.

Actions taken:

* Verified node operation.
* Continued regular updates.
* Maintained node availability.

Result:

* Reputation recovered to 5000.
